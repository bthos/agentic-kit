# Memory Schema

This file is the **ontology** for `.agentic-kit-artefacts/memory/`. Every entry across the L2/L3 layers follows this shape so `memory-search.sh`, `memory-promote.sh`, and the agents that read memory can rely on a stable contract.

## Layered storage

| Layer | Path | What lives here |
|-------|------|-----------------|
| **L0 — Enforcement** | `agents/*.md`, `skills/*/SKILL.md`, `agentic-kit/autoresearch/program.md` | Hardened rules. Mutated only via `apply-patches.sh` or Veles ratchet. |
| **L1 — Hot State** | `.agentic-kit-artefacts/SESSION-STATE.md` | Current active feature, current agent, in-flight decisions. Reset / pruned aggressively. |
| **L2 — Daily Memory** | `.agentic-kit-artefacts/memory/YYYY-MM-DD.md` | Per-day rolling log of observations, lessons, decisions. Append-only; rolled over by `memory-rollover.sh`. |
| **L3 — Long-term structured** | `.agentic-kit-artefacts/memory/{preferences,system,projects,decisions}.md` | Curated, deduplicated facts grouped by entity type. |
| **L4 — Root summary** | `.agentic-kit-artefacts/MEMORY.md` | ≤2 KB index pointing to L3 sections. Read first by every skill. |
| **L5 — Semantic recall** | `.agentic-kit-artefacts/memory/index.jsonl` | Chunked text + lightweight retrieval surface for `memory-search.sh`. Auto-rebuilt. |

## Entity types (fixed)

Every entry MUST declare `entity_type:` from this set. This is the answer to OpenClaw's "no custom ontology" gap.

- `person` — humans (user, teammate, contributor)
- `project` — codebase, feature, subsystem
- `file` — concrete file path
- `tool` — CLI, build tool, framework, lib
- `library` — npm/pip/cargo dep
- `pattern` — design / coding pattern, naming, convention
- `decision` — explicit ADR-style decision
- `anti-pattern` — known failure mode

## Entry shape (L2 and L3)

Every memory entry is a markdown bullet block of this exact form:

```
- id: mem_<short-hash>
  decided: 2026-04-30
  entity_type: pattern
  entities: [<name1>, <name2>]
  confidence: high      # high | medium | low
  supersedes: mem_<id>  # OPTIONAL — points to entry this one replaces
  source: archive/<feature-id>/LESSONS.md   # OPTIONAL
  text: |
    One- or two-line concrete fact. Specific, actionable, verifiable.
```

Rules:

1. **`id` is content-addressed**: SHA-1 of the `text` field, truncated to 8 chars, prefixed `mem_`. Computed by `memory-promote.sh`. Never edit by hand.
2. **`decided` is mandatory** for L3. Allows the **temporal-awareness** resolver (OpenClaw gap #1) to deprioritise older entries when newer ones supersede them.
3. **`supersedes`** is the explicit replacement pointer. When present, `memory-search.sh` returns the new entry but tags the old one as `[superseded]` (instead of silently dropping it — Навь principle).
4. **`entities`** is a flat list. Lets simple `grep`-traversal find related entries (OpenClaw gap #2: no relationships) without needing a graph DB.
5. **`confidence`** drives the agent's behaviour: `high` is treated as a rule; `medium` as advisory; `low` is reference only.

## Promotion state machine

```
observed → logged (L2 daily) → curated (L3) → hardened (L0 agent prompt) → stable
```

Implemented by `memory-promote.sh`:

- **observed → logged** — agents append to today's L2 file as they work (see "Mandatory write checklist" in each agent prompt).
- **logged → curated** — when the same fact appears in **2 different daily files** (the **2-strike rule**), it auto-moves to the right L3 file with `confidence: medium`.
- **curated → hardened** — when an L3 entry has `confidence: high` AND has been referenced in 3+ subsequent features without contradiction, `memory-promote.sh --propose-hardening` writes a patch to `.agentic-kit-artefacts/proposed-patches/<agent>.md` for review with `tools/apply-patches.sh`.
- **stable** — the rule lives in the agent prompt; the L3 entry is kept as evidence (Навь) with a `hardened_in: agent.md@<sha>` tag.

## Layer 4 (root summary) format

`.agentic-kit-artefacts/MEMORY.md` is **regenerated** by `memory-promote.sh` on every promote run. Hand-edits will be overwritten — capture intent in L3 instead.

```
# Memory Index

_Generated by memory-promote.sh. Do not edit by hand — edit the linked layer files._

## High-confidence preferences
- → `memory/preferences.md` (N entries)
…

## High-confidence system facts
- → `memory/system.md` (N entries)
…

## Most-referenced decisions (last 90 days)
- mem_abcd1234 — <text>
…

## Recent supersessions
- mem_aaaa1111 superseded by mem_bbbb2222 on 2026-04-30
…
```

## How agents must use memory

1. **Read first.** Before any task, read `.agentic-kit-artefacts/MEMORY.md`. Drill into L3 only when needed.
2. **Search when uncertain.** `agentic-kit/tools/memory-search.sh "<query>"` returns top-k chunks across all layers.
3. **Write when you learn.** Every agent has a "Mandatory write checklist" — append to today's L2 file when any trigger fires (new convention, new tool, anti-pattern, decision).
4. **Never edit L4.** It's regenerated.
5. **Never delete L2/L3 entries.** Use `supersedes:` instead. Навь is preserved.
